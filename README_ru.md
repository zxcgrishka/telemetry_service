# Сервис Телеметрии

[English Version](README.md)

RESTful API сервис для сбора телеметрических данных с устройств и асинхронной обработки аналитики.

## Основные возможности
- **Полноценная JWT Аутентификация:** Использование Access и Refresh токенов для безопасных сессий.
- **UUID Идентификаторы:** Устройства используют стандартные UUID вместо простых чисел.
- **Асинхронная Аналитика:** Тяжелые расчеты метрик выполняются в фоне с помощью Celery и Redis.
- **Пагинация данных:** Высокопроизводительное получение телеметрии с поддержкой limit/skip и фильтрацией по времени.

## Технологический стек
- **Фреймворк:** FastAPI
- **База данных:** PostgreSQL
- **Очередь задач:** Celery + Redis
- **Контейнеризация:** Docker и Docker Compose
- **Нагрузочное тестирование:** Locust

## Структура проекта
- `app/`: Основной код приложения.
  - `main.py`: Точка входа FastAPI и подключение роутеров.
  - `models.py`: Модели базы данных SQLAlchemy (PostgreSQL).
  - `schemas.py`: Схемы данных Pydantic (паттерн DTO).
  - `crud.py`: Операции с базой данных (создание, чтение, поиск).
  - `worker.py`: Задачи и конфигурация фоновых воркеров Celery.
  - `config.py`: Настройки приложения и управление переменными окружения.
  - `database.py`: Настройка движка БД и управление сессиями.
  - `deps.py`: Зависимости FastAPI (JWT аутентификация).
  - `security.py`: Утилиты безопасности (создание JWT, хеширование паролей).
  - `routers/`: API эндпоинты, сгруппированные по ресурсам.
- `alembic/`: Скрипты и среда миграций базы данных.
- `alembic.ini`: Конфигурация для миграций Alembic.
- `Dockerfile` и `docker-compose.yml`: Настройки контейнеризации и оркестрации.
- `pyproject.toml` и `poetry.lock`: Зависимости Python и метаданные проекта.
- `locustfile.py`: Сценарий нагрузочного тестирования.
- `locust_report.html`: Подробный HTML-отчет последнего теста.

## Как запустить

### Использование Docker (Рекомендуется)
1. Убедитесь, что у вас установлены и запущены Docker и Docker Compose.
2. Выполните команду:
   ```bash
   docker-compose up --build
   ```
3. API будет доступно по адресу `http://localhost:8080`.
4. Интерактивная документация (Swagger UI) доступна по адресу `http://localhost:8080/docs`.

### Локальная разработка (без Docker)
1. Установите зависимости через Poetry:
   ```bash
   poetry install
   ```
2. Поднимите PostgreSQL и Redis.
3. Настройте переменные окружения (через export или настройки системы).
   **Примечание:** При развертывании в продакшн (deploy) настоятельно рекомендуется перенести все секретные данные (пароли, ключи) в файл `.env`, который должен быть исключен из системы контроля версий.
   ```
   DATABASE_URL=postgresql+asyncpg://user:pass@localhost/telemetry
   SYNC_DATABASE_URL=postgresql://user:pass@localhost/telemetry
   REDIS_URL=redis://localhost:6379/0
   ```
4. Примените миграции:
   ```bash
   poetry run alembic upgrade head
   ```
5. Запустите сервер:
   ```bash
   poetry run uvicorn app.main:app --reload --port 8080
   ```
6. Запустите воркер Celery:
   ```bash
   poetry run celery -A app.worker.celery worker --loglevel=info
   ```

## API Эндпоинты

### Пользователи и Аутентификация
- `POST /users/`: Регистрация нового пользователя.
- `POST /users/login`: Получение Access и Refresh токенов.
- `POST /users/refresh`: Обновление Access токена через Refresh токен.

### Устройства
- `GET /devices/`: Список всех устройств текущего пользователя.
- `POST /devices/`: Регистрация нового устройства (UUID генерируется автоматически).

### Сбор телеметрии
- `GET /telemetry/{device_id}`: Получение истории с пагинацией и фильтрами по времени.
- `POST /telemetry/{device_id}`: Прием данных телеметрии (`x`, `y`, `z`).

### Аналитика
- `POST /analytics/device/{device_id}`: Запуск анализа устройства. Возвращает `task_id`.
- `POST /analytics/user/{user_id}`: Запуск анализа по всем устройствам пользователя.
- `GET /analytics/task/{task_id}`: Проверка статуса и получение результатов (min, max, median и т.д.).

## Нагрузочное тестирование
Результаты сохранены в файле `locust_report.html`. Для ручного запуска:
1. Запустите сервис.
2. Выполните `poetry run locust`.
3. Откройте `http://localhost:8089`.
